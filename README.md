# Washington State Influenza Database and Pipeline  

This project builds an automated ETL pipeline to collect, clean, and organize influenza data from multiple sources, including live CDC FluView APIs.  
The pipeline integrates county- and state-level metrics into a PostgreSQL database for tracking and analysis of influenza trends across Washington State.  

## Features  
- Automated data extraction and cleaning in Python  
- Workflow orchestration with Apache Airflow  
- Relational database schema and ERD design in PostgreSQL  
- Geographic and temporal trend analysis  

## Tools & Technologies  
Python | Airflow | PostgreSQL | Pandas | SQL | Flask  

## Status  
Developed as part of *Data Engineering Principles and Practices* (Fall 2025). Future work includes dashboard integration and predictive modeling.
